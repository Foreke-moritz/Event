Security at CoreInventory

Your business data is our top priority. Here's how we keep it safe.

Authentication & Access Control

CoreInventory uses industry-standard authentication powered by Supabase Auth, ensuring your credentials are never stored in plain text.

  • Secure password hashing using bcrypt algorithms
  • Role-based access control (Admin vs Staff) enforced at both application and database level
  • Session tokens with automatic expiration and refresh
  • Password reset via secure email verification

Data Isolation & Row-Level Security

Every business on CoreInventory is completely isolated. No business can ever access another business's data.

  • PostgreSQL Row-Level Security (RLS) policies active on all tables
  • Each record is tied to a specific admin_id for strict ownership
  • Staff members can only access data belonging to their assigned admin
  • RLS is enforced at the database engine level — bypassing is impossible from the application layer

Cloud Infrastructure

CoreInventory runs on enterprise-grade cloud infrastructure provided by Supabase, built on top of AWS.

  • All data transmitted over HTTPS/TLS encryption
  • Automated daily database backups
  • 99.9% uptime SLA guarantee
  • Data hosted in secure, SOC 2 compliant data centers

Your Responsibilities

Security is a shared responsibility. Here are some best practices we recommend:

  • Use a strong, unique password for your CoreInventory account
  • Never share your admin credentials with staff — create staff accounts instead
  • Log out from shared or public devices after each session
  • Report any suspicious activity to our support team immediately
